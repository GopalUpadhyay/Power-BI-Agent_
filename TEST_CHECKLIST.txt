COMMON ISSUES & QUICK FIXES:
═════════════════════════════════════════════════════════════════════════

Issue: At Step 7, files not appearing in upload widget
Fix:
  □ Close file browser and try again
  □ Make sure files are .csv format
  □ Check file names: SalesPerson.csv, Sales.csv (exact names)
  □ Browser cache - Ctrl+Shift+Delete to clear cache


Issue: At Step 8, "Identified 0 relationships"
Fix:
  □ Check if EmployeeID column exists in BOTH files
  □ Click "🔁 Re-Detect Relationships" to try again
  □ Manually create relationship if auto-detect fails
     → Add row in table: from_table=Sales, from_column=EmployeeID,
       to_table=SalesPerson, to_column=EmployeeID


Issue: At Step 11, Error when saving relationships
Fix:
  □ Make sure all 4 fields are filled in:
     - from_table
     - from_column
     - to_table
     - to_column
  □ No empty cells allowed
  □ Try re-entering data


Issue: Model not appearing after creation
Fix:
  □ Scroll down to see "Available Model Cards"
  □ Click browser refresh button (Ctrl+R)
  □ Or click "🔄 Reload Active Model" button
